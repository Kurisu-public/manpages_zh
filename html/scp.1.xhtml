名称

scp — 安全复制 (远程文件复制程序)

总览 (SYNOPSIS)

scp [-12346BCpqrv] [-c cipher] [-F ssh_config] [-i identity_file] [-l limit] [-o ssh_option] [-P port] [-S program] [[user@]host1:]file1 ... [[user@]host2:]file2

描述 (DESCRIPTION)

scp 在网络的主机间复制文件。 它使用 ssh(1) 实现数据传输, 使用相同的验证方式, 提供与 ssh(1) 同等的安全性。 scp 会在有必要的情况下 向用户询问密码与验证密文 (passphrase) 用来进行身份验证。

文件名可能包含一个用户名和主机信息, 以此指示文件将被/从该主机复制。 本地文件名称可以显式地使用绝对或相对路径名, 以此避免 scp 将含有 `:' 字符的文件名字符串当作主机信息处理。 在两台远程主机间进行文件复制同样是被允许的。

选项 (OPTIONS)

可接受的选项如下所示：

-1

强制 scp 使用协议 1。

-2

强制 scp 使用协议 2。

-3

在两台远程主机之间复制文件时经由本地主机中转。 如果未指定该选项, 数据将在两台远程主机之间直接传输。 请注意该选项将禁用进度条显示。

-4

强制 scp 仅使用 IPv4 地址。

-6

强制 scp 仅使用 IPv6 地址。

-B

选择 batch 模式 (阻止对密码或验证密文的询问)。

-C

启用压缩。 向 ssh(1) 传递 -C 参数以启用压缩。

-c cipher

选择用来加密数据传输的 cipher。 该选项参数将直接传递至 ssh(1)。

-F ssh_config

显式指定用于 ssh 的用户配置文件。 该选项参数将直接传递至 ssh(1)。

-i identify_file

选择用于公钥身份验证的身份 (私钥) 文件。 该选项参数将直接传递至 ssh(1)。

-l limit

限制使用带宽, 单位为千字节每秒 (Kbit/s)。

-o ssh_option

可被用来将 ssh_config(5) 格式的参数传递给 ssh。 该选项在指定那些 scp 缺少的 ssh 命令行参数时尤为有用。 如需了解下列支持选项的完整详细说明, 以及它们的可能取值, 请参阅 ssh_config(5)。

-P port

指定连接远程主机的端口号。 请注意该选项使用大写 `P', 因为 -p 已被用来指定保留文件的时间与模式信息。

-p

保留文件修改时间, 访问时间, 以及源文件的模式 (modes)。

-q

安静模式; 不显示由 ssh(1) 传来的进度条、警告、诊断信息。

-r

递归地复制整个目录。 请注意, scp 跟随遍历过程中遇到的符号链接。

-S program

指定用来进行加密连接的 program (程序) 名称。 该程序必须能够识别 ssh(1) 的参数与选项。

-v

详述 (Verbose) 模式。 使得 scp 和 ssh(1) 打印执行过程中的调试信息。 该选项在调试连接、授权以及配置问题时十分有用。

返回状态 (EXIT STATUS)

Ex -std scp
